# ResourceSharing Domain Reference

## Overview

The `ResourceSharing` domain class provides a Python interface for managing resource sharing operations in Alma through the Partners API. It handles lending requests, which represent requests from partner institutions to borrow materials from your library.

### When to Use

- Creating lending requests from external ILL systems (Rapido, ILLiad, Relais)
- Automating interlibrary loan workflows
- Retrieving and monitoring lending request statuses
- Integrating with citation metadata services (PubMed, Crossref)

### Key Concepts

| Concept | Description |
|---------|-------------|
| **Partner** | An institution you share resources with, identified by a partner code |
| **Lending Request** | A request FROM a partner TO borrow material from your library |
| **External ID** | Your system's tracking identifier for the request (e.g., ILL transaction number) |
| **Owner** | The resource sharing library code that will fulfill the request |
| **Citation Metadata** | Bibliographic information auto-populated from PubMed or Crossref |

## Methods Reference

### create_lending_request()

Creates a new lending request for a partner institution.

#### Signature

```python
def create_lending_request(
    self,
    partner_code: str,
    external_id: str,
    owner: str,
    format_type: str,
    title: str,
    citation_type: Optional[str] = None,
    mms_id: Optional[str] = None,
    **optional_fields
) -> Dict[str, Any]
```

#### Parameters

**Required Parameters:**

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `partner_code` | `str` | Partner institution code | `"RELAIS"` |
| `external_id` | `str` | Your external tracking ID | `"ILL-2025-001"` |
| `owner` | `str` | Resource sharing library code | `"MAIN"` |
| `format_type` | `str` | Request format | `"PHYSICAL"` or `"DIGITAL"` |
| `title` | `str` | Resource title (required unless `mms_id` provided) | `"Introduction to Library Science"` |

**Conditional Parameters:**

| Parameter | Type | Description | When Required |
|-----------|------|-------------|---------------|
| `citation_type` | `str` | Resource type (BOOK, JOURNAL, etc.) | Required if no `mms_id` |
| `mms_id` | `str` | Alma catalog record ID | Replaces `citation_type` and `title` requirements |

**Optional Parameters (via `**optional_fields`):**

| Parameter | Type | Description |
|-----------|------|-------------|
| `author` | `str` | Resource author |
| `isbn` | `str` | ISBN for books |
| `issn` | `str` | ISSN for journals |
| `publisher` | `str` | Publisher name |
| `year` | `str` | Publication year |
| `edition` | `str` | Edition information |
| `volume` | `str` | Volume number |
| `issue` | `str` | Issue number |
| `pages` | `str` | Page range |
| `doi` | `str` | Digital Object Identifier |
| `pmid` | `str` | PubMed ID |
| `call_number` | `str` | Library call number |
| `oclc_number` | `str` | OCLC number |
| `level_of_service` | `dict` or `str` | Service level (e.g., "Rush") |
| `last_interest_date` | `str` | Need-by date (ISO format: `"2025-02-01Z"`) |
| `rs_note` | `list` | Notes array |

#### Return Value

Returns a dictionary containing the created lending request with all fields, including:

```python
{
    "request_id": "35889547910004146",  # Generated by Alma
    "external_id": "ILL-2025-001",
    "status": {"value": "REQUEST_CREATED_LEN", "desc": "Request Created"},
    "owner": "MAIN",
    "partner": {"value": "RELAIS", "desc": "RELAIS Partner"},
    "format": {"value": "PHYSICAL", "desc": "Physical"},
    "title": "Introduction to Library Science",
    # ... additional fields
}
```

#### Raises

| Exception | Condition |
|-----------|-----------|
| `ValueError` | Missing or invalid mandatory fields |
| `AlmaAPIError` | API request failure (authentication, network, etc.) |

**Example 1: Basic Book Request**

| Error | Cause | Solution |
|-------|-------|----------|

### create_lending_request_from_citation()

Creates a lending request with metadata auto-populated from PubMed or Crossref.

#### Signature

```python
def create_lending_request_from_citation(
    self,
    partner_code: str,
    external_id: str,
    owner: str,
    format_type: str,
    pmid: Optional[str] = None,
    doi: Optional[str] = None,
    source_type: Optional[str] = None,
    **override_fields
) -> Dict[str, Any]
```

#### Parameters

**Required Parameters:**

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `partner_code` | `str` | Partner institution code | `"RELAIS"` |
| `external_id` | `str` | Your external tracking ID | `"ILL-2025-001"` |
| `owner` | `str` | Resource sharing library code | `"MAIN"` |
| `format_type` | `str` | Request format | `"PHYSICAL"` or `"DIGITAL"` |

**Identifier Parameters (at least one required):**

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `pmid` | `str` | PubMed ID | `"33219451"` |
| `doi` | `str` | Digital Object Identifier | `"10.1038/s41591-020-1124-9"` |

**Optional Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `source_type` | `str` | Explicit source: `'pmid'` or `'doi'`. If specified, only that source is tried (no fallback). Recommended for form inputs. |
| `**override_fields` | `dict` | Fields to override auto-populated values |

#### Return Value

Returns the created lending request dictionary (same as `create_lending_request`).

Auto-populated fields include:
- `title`, `author`, `journal_title`, `publisher`
- `year`, `volume`, `issue`, `pages`
- `doi`, `pmid`, `issn`

#### Raises

| Exception | Condition |
|-----------|-----------|
| `ValueError` | Neither pmid nor doi provided, or invalid source_type |
| `CitationMetadataError` | Metadata fetch from PubMed/Crossref failed |
| `AlmaAPIError` | API request failure |

#### Code Examples

**Example 1: Explicit PMID Source (Recommended)**

```python
# Use when you know the identifier type (e.g., from a form)
request = rs.create_lending_request_from_citation(
    partner_code="RELAIS",
    external_id="ILL-2025-001",
    owner="MAIN",
    format_type="DIGITAL",
    pmid="33219451",
    source_type='pmid'  # Only try PubMed, no fallback
)

print(f"Title: {request['title']}")  # Auto-populated from PubMed
print(f"Journal: {request.get('journal_title')}")
```

**Example 2: Explicit DOI Source**

```python
request = rs.create_lending_request_from_citation(
    partner_code="RELAIS",
    external_id="ILL-2025-002",
    owner="MAIN",
    format_type="DIGITAL",
    doi="10.1038/s41591-020-1124-9",
    source_type='doi'  # Only try Crossref
)
```

**Example 3: Auto-detect Mode with Fallback**

```python

# then falls back to Crossref (if doi provided)

    partner_code="RELAIS",
    external_id="ILL-2025-003",
    owner="MAIN",
    format_type="DIGITAL",
    pmid="33219451",
    doi="10.1038/s41591-020-1124-9"  # Fallback if PubMed fails

```

**Example 4: Override Auto-populated Fields**

```python

    partner_code="RELAIS",
    external_id="ILL-2025-004",
    owner="MAIN",
    format_type="PHYSICAL",
    doi="10.1038/s41591-020-1124-9",
    source_type='doi',
    title="Custom Title Override",  # Override auto-fetched title
    pages="100-150"  # Override auto-fetched pages
)
```

#### Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `ValueError: Must provide at least one of: pmid, doi` | Neither identifier provided | Provide pmid or doi |
| `CitationMetadataError: PubMed fetch failed` | Invalid PMID or network error | Verify PMID, check network |
| `CitationMetadataError: Crossref fetch failed` | Invalid DOI or network error | Verify DOI format |
| `CitationMetadataError: Failed to fetch metadata from all sources` | Both sources failed | Check identifiers and network |

### Workflow 1: Create Lending Request

The standard workflow for creating a lending request from an external ILL system.

```python
from almaapitk import AlmaAPIClient, ResourceSharing, AlmaAPIError

def create_ill_request(ill_data):
    """Process an ILL request from external system."""

    client = AlmaAPIClient('PRODUCTION')
    rs = ResourceSharing(client)

    try:
        request = rs.create_lending_request(
            partner_code=ill_data['partner'],
            external_id=ill_data['transaction_id'],
            owner=ill_data['library'],
            format_type=ill_data['format'],
            title=ill_data['title'],
            citation_type=ill_data['material_type'],
            author=ill_data.get('author'),
            isbn=ill_data.get('isbn'),
            issn=ill_data.get('issn'),
            volume=ill_data.get('volume'),
            issue=ill_data.get('issue'),
            pages=ill_data.get('pages')
        )

        return {
            'success': True,
            'alma_request_id': request['request_id'],
            'status': request['status']['value']
        }

    except ValueError as e:
        # Validation error - data issue
        return {'success': False, 'error_type': 'VALIDATION', 'error': str(e)}

    except AlmaAPIError as e:
        # API error - Alma issue
        return {'success': False, 'error_type': 'API', 'error': str(e)}
```

### Workflow 2: Create Request with Citation Metadata (PubMed/Crossref)

Automatically populate bibliographic data from external sources.

```python
from almaapitk import AlmaAPIClient, ResourceSharing
from almaapitk.utils.citation_metadata import CitationMetadataError

def create_request_with_metadata(external_id, identifier, identifier_type, owner="MAIN"):
    """Create request with auto-populated metadata from PubMed or Crossref."""

    client = AlmaAPIClient('SANDBOX')
    rs = ResourceSharing(client)

    try:
        # Prepare identifier arguments
        kwargs = {
            'partner_code': 'RELAIS',
            'external_id': external_id,
            'owner': owner,
            'format_type': 'DIGITAL'
        }

        # Set identifier based on type
        if identifier_type == 'pmid':
            kwargs['pmid'] = identifier
            kwargs['source_type'] = 'pmid'
        elif identifier_type == 'doi':
            kwargs['doi'] = identifier
            kwargs['source_type'] = 'doi'
        else:
            raise ValueError(f"Unknown identifier type: {identifier_type}")

        # Create request with metadata enrichment
        request = rs.create_lending_request_from_citation(**kwargs)

        print(f"Created request: {request['request_id']}")
        print(f"Title (auto-populated): {request['title']}")
        print(f"Author (auto-populated): {request.get('author', 'N/A')}")

        return request

    except CitationMetadataError as e:
        print(f"Failed to fetch metadata: {e}")
        # Fall back to manual entry or handle error
        raise
    except AlmaAPIError as e:
        print(f"API error: {e}")
        raise

# Usage examples
request = create_request_with_metadata(
    external_id="ILL-2025-001",
    identifier="33219451",
    identifier_type="pmid"
)

request = create_request_with_metadata(
    external_id="ILL-2025-002",
    identifier="10.1038/s41591-020-1124-9",
    identifier_type="doi"
)
```

## Best Practices and Gotchas

### 1. Owner Field Format

**CRITICAL**: The `owner` field must be a plain string, NOT a wrapped object. The official Alma schema documentation is incorrect.

```python
# CORRECT - Plain string
request = rs.create_lending_request(
    owner="MAIN",  # Plain string
    ...
)

# INCORRECT - Schema docs show this but it causes 400 error!
# {"owner": {"value": "MAIN"}}  # DO NOT USE
```

### 2. Partner Code vs Display Name

Always use the internal code, not the display name:

```python
# Response structure
partner = request['partner']
# {
#     "value": "ANC",        # Use THIS in API calls
#     "desc": "ANCA-TEST"    # Display name - don't use for API
# }

# Correct
rs.get_lending_request(partner_code="ANC", request_id="...")

# Wrong - will fail
rs.get_lending_request(partner_code="ANCA-TEST", request_id="...")
```

### 3. Unique External IDs

Always provide unique external identifiers:

```python
# Good - unique identifier
external_id = f"ILL-{system_id}-{timestamp}"

# Bad - may cause tracking issues
external_id = "REQUEST-1"
```

### 4. Date Format

Use ISO 8601 format with 'Z' suffix for dates:

```python
# Correct
last_interest_date = "2025-02-28Z"

# Wrong - will fail
last_interest_date = "02/28/2025"
```

### 5. Undocumented Fields

Real API responses include useful fields not in schema documentation:

```python
request = rs.get_lending_request("RELAIS", "12345678")

# Undocumented but useful fields
barcode = request.get('barcode')  # Physical item barcode
text_email = request.get('text_email')  # Contact email
created_time = request.get('created_time')  # Precise timestamp (milliseconds)
has_active_notes = request.get('has_active_notes')  # Quick check for notes
```

### 6. Test in SANDBOX First

Always validate new integrations in SANDBOX before PRODUCTION:

```python
# Development/Testing
client = AlmaAPIClient('SANDBOX')

# Production (after thorough testing)
client = AlmaAPIClient('PRODUCTION')
```

## Error Reference

### Validation Errors (ValueError)

| Error Message | Cause | Solution |
|---------------|-------|----------|
| `external_id is mandatory when creating a lending request` | Missing or empty external_id | Provide unique identifier |
| `owner is mandatory for lending requests` | Missing owner field | Provide library code as plain string |
| `partner is mandatory when creating a lending request` | Missing partner code | Provide partner_code parameter |
| `format is mandatory` | Missing format | Provide format_type (PHYSICAL/DIGITAL) |
| `citation_type is mandatory when creating a lending request unless mms_id is supplied` | Neither citation_type nor mms_id | Provide one of them |
| `title is mandatory unless mms_id is supplied` | Title missing without mms_id | Provide title or mms_id |

### API Errors (AlmaAPIError)

| Status Code | Meaning | Solution |
|-------------|---------|----------|
| 400 | Bad request (e.g., invalid partner code) | Verify partner exists in Alma configuration |
| 401 | Authentication failed | Check API key environment variable |
| 404 | Request not found | Verify request_id and partner_code match |
| 500 | Server error | Retry later, contact support if persists |

### Citation Metadata Errors (CitationMetadataError)

| Error | Cause | Solution |
|-------|-------|----------|
| `PubMed API request timed out` | Network timeout | Retry or check network |
| `Article not found for PMID` | Invalid PMID | Verify PMID is correct |
| `DOI not found` | Invalid DOI | Verify DOI format and existence |
| `Failed to fetch metadata from all sources` | Both sources failed | Check identifiers and network |

### Workflow 3: ILL System Integration

Integrate with external ILL systems (Rapido, ILLiad, etc.) with field mapping:

```python
def process_ill_requests(ill_requests):
    """Process batch of ILL requests from external system."""

    client = AlmaAPIClient('PRODUCTION')
    rs = ResourceSharing(client)

    results = []

    for ill_req in ill_requests:
        try:
            request = rs.create_lending_request(
                partner_code=ill_req['partner_code'],
                external_id=ill_req['transaction_id'],
                owner=ill_req['owning_library'],
                format_type=map_format(ill_req['format']),
                title=ill_req['title'],
                citation_type=map_citation_type(ill_req['material_type']),
                author=ill_req.get('author'),
                isbn=ill_req.get('isbn'),
                issn=ill_req.get('issn'),
                volume=ill_req.get('volume'),
                issue=ill_req.get('issue'),
                pages=ill_req.get('pages')
            )

            results.append({
                'ill_id': ill_req['transaction_id'],
                'alma_id': request['request_id'],
                'status': 'SUCCESS'
            })

        except (ValueError, AlmaAPIError) as e:
            results.append({
                'ill_id': ill_req['transaction_id'],
                'status': 'FAILED',
                'error': str(e)
            })

    return results

def map_format(ill_format):
    """Map ILL format to Alma format."""
    mapping = {
        'loan': 'PHYSICAL',
        'copy': 'DIGITAL',
        'physical': 'PHYSICAL',
        'electronic': 'DIGITAL'
    }
    return mapping.get(ill_format.lower(), 'PHYSICAL')

def map_citation_type(material_type):
    """Map ILL material type to Alma citation type."""
    mapping = {
        'book': 'BOOK',
        'journal': 'JOURNAL',
        'article': 'JOURNAL',
        'thesis': 'THESIS',
        'conference': 'BOOK'
    }
    return mapping.get(material_type.lower(), 'BOOK')
```

```python
def monitor_requests(partner_code, request_ids):
    """Monitor status of multiple requests."""

    client = AlmaAPIClient('PRODUCTION')
    rs = ResourceSharing(client)

    status_report = []

    for req_id in request_ids:
        try:
            request = rs.get_lending_request(partner_code, req_id)
            summary = rs.get_request_summary(request)

            status_report.append({
                'request_id': req_id,
                'title': summary['title'],
                'status': summary['status'],
                'format': summary['format']
            })

            # Take action based on status
            if summary['status'] == "REQUEST_CREATED_LEN":
                notify_staff("New request awaiting processing", req_id)
            elif summary['status'] == "SHIPPED_PHYSICALLY":
                notify_partner("Item shipped", req_id)

        except AlmaAPIError as e:
            status_report.append({
                'request_id': req_id,
                'status': 'ERROR',
                'error': str(e)
            })

    return status_report
```

## Alma Configuration Requirements

Before using the ResourceSharing domain, ensure Alma is configured:

1. **Partner Codes**
   - Path: Configuration > Resource Sharing > Partners
   - Add partner institutions with unique codes

2. **Resource Sharing Libraries**
   - Path: Configuration > General > Libraries
   - Enable resource sharing for relevant libraries
   - Set library codes (used as `owner` parameter)

3. **Format Code Table**
   - Path: Configuration > Resources > Resource Sharing
   - Verify RequestFormats values (PHYSICAL, DIGITAL)

4. **Citation Type Code Table**
   - Verify citation type values (BOOK, JOURNAL, THESIS, etc.)

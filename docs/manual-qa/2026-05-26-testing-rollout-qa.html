How this works

Work the cards below: set a status, jot notes, and hit 💬 Ping Claude on any card when you want me to act on it. Your ping (card + status + notes) reaches me, I do the work and reply inline in that card. Statuses/notes persist locally; when the server is running they also sync to me.

Each card is one concrete step. Sections are grouped by repo. Anything tagged read-only is safe; mutating needs care (teardown/no-writes-to-prod).

Strategy at a glance

Infra — the harness toolkit in almaapitk[smoke] (read-only rail, dry-run, retry). Shipped, unreleased.
L1 — Contract tests — in almaapitk; pin behaviours consumers rely on; run every change. Analytics done.
L2 — Mock/golden — per repo, offline. Analytics has it.
L3 — Live smoke — per repo, real read-only Alma; the masedet replacement.

Per-repo loop: audit usage → add L1 contract tests (almaapitk) → self-contained live test (repo) → run on masedet → bump prod to 0.4.5. Adopt the harness later, only where it earns its keep (mutating repos).
